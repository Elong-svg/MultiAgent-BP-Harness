本文档详细描述 business-plan-creator Skill 的完整执行流程，包含5个Phase的标准化操作步骤。

## Phase 1: 需求确认与数据收集

### 输入
- 用户提供的公司名称或股票代码
- 可选：特定分析要求、输出格式偏好

### 执行步骤

#### Step 1.1: 确认需求

**与用户确认以下信息：**

| 确认项 | 说明 | 示例 |
|--------|------|------|
| 公司名称 | 完整公司中文名称 | 海尔智家股份有限公司 |
| 股票代码 | 交易所代码 | 600690.SH / 06690.HK |
| 输出格式 | Word/PDF | PDF（推荐用于正式交付） |
| 分析深度 | 基础/标准/深度 | 标准（适用于大多数场景） |
| 特殊要求 | 定制化需求 | 侧重海外市场分析等 |

**调用指令：**
```
使用 ask_followup_question 工具收集用户确认信息
```

#### Step 1.2: 调用 finance-data-retrieval 获取数据

**2.1 获取公司基本信息**
```
调用: finance-data-retrieval
参数:
  - api: stock_basic
  - ts_code: [股票代码]
输出: 公司名称、行业、市值、成立时间等
```

**2.2 获取财务报表数据**
```
调用: finance-data-retrieval
参数:
  - api: income（利润表）
  - api: balancesheet（资产负债表）  
  - api: cashflow（现金流量表）
  - ts_code: [股票代码]
  - period: 最近5个财年
输出: 完整财务报表数据
```

**2.3 获取行业数据**
```
调用: finance-data-retrieval
参数:
  - api: 行业相关API
  - industry: [公司所属行业]
输出: 行业规模、增长率、关键指标
```

**2.4 获取竞争对手数据**
```
调用: finance-data-retrieval
参数:
  - api: stock_basic（获取同行业公司列表）
  - 对前3-5家竞争对手获取财务数据
输出: 竞争对手关键财务指标对比数据
```

#### Step 1.3: 数据验证

**检查清单：**

- [ ] 数据完整性检查（最近5年数据是否完整）
- [ ] 异常值检查（识别明显错误数据）
- [ ] 缺失值标注（记录缺失数据的年份/科目）
- [ ] 数据来源标注（所有数据标注来源和获取时间）

**处理缺失数据策略：**
1. 单一缺失：使用前后年份数据插值
2. 多期缺失：标注"数据不可得"，不强行估算
3. 关键指标缺失：降低该维度分析权重

### 输出

**数据集结构：**
```
/data/
├── company_info.json       # 公司基本信息
├── financial_statements/
│   ├── income.json         # 利润表（5年）
│   ├── balance_sheet.json  # 资产负债表（5年）
│   └── cashflow.json       # 现金流量表（5年）
├── industry_data.json      # 行业数据
├── competitors/
│   ├── comp_1.json         # 竞争对手1数据
│   ├── comp_2.json
│   └── comp_3.json
└── data_source_notes.md    # 数据来源标注文档
```

## Phase 2: 财务分析

### 输入
- Phase 1 获取的完整数据集

### 执行步骤

#### Step 2.1: 调用 finance 技能进行财务指标分析

**2.1.1 盈利能力分析**
```
调用: finance
任务: 盈利能力分析
参数:
  - 输入数据: income.json, balance_sheet.json
  - 计算指标: ROE, ROA, 毛利率, 净利率, EPS
  - 分析维度: 5年趋势、同业对比
输出: 盈利能力分析报告
```

**2.1.2 偿债能力分析**
```
调用: finance
任务: 偿债能力分析
参数:
  - 输入数据: balance_sheet.json
  - 计算指标: 流动比率, 速动比率, 资产负债率, 利息保障倍数
输出: 偿债能力分析报告
```

**2.1.3 营运能力分析**
```
调用: finance
任务: 营运能力分析
参数:
  - 输入数据: income.json, balance_sheet.json
  - 计算指标: 存货周转率, 应收账款周转率, 总资产周转率
输出: 营运能力分析报告
```

**2.1.4 成长能力分析**
```
调用: finance
任务: 成长能力分析
参数:
  - 输入数据: income.json, balance_sheet.json
  - 计算指标: 营收增长率, 净利润增长率, 总资产增长率
输出: 成长能力分析报告
```

**2.1.5 现金流分析**
```
调用: finance
任务: 现金流分析
参数:
  - 输入数据: cashflow.json
  - 计算指标: 经营现金流, 自由现金流, 现金流比率
输出: 现金流分析报告
```

#### Step 2.2: 调用 web_search 技能进行外部环境分析

**2.2.1 行业趋势分析**
```
调用: web_search
查询: "[行业名称] 发展趋势 2024 2025"
输出: 行业最新动态、技术趋势、政策影响
```

**2.2.2 竞争对手分析**
```
调用: web_search
查询: "[竞争对手名称] 2024 年报 业绩"
输出: 竞争对手最新战略、市场表现
```

**2.2.3 宏观环境分析**
```
调用: web_search
查询: "[相关行业] 宏观经济 政策影响 2024"
输出: 宏观经济指标、政策变化、市场风险
```

### 输出

**分析报告结构：**
```
/analysis/
├── profitability_analysis.md    # 盈利能力分析
├── solvency_analysis.md         # 偿债能力分析
├── operation_analysis.md        # 营运能力分析
├── growth_analysis.md           # 成长能力分析
├── cashflow_analysis.md         # 现金流分析
├── industry_analysis.md         # 行业趋势分析
├── competitor_analysis.md       # 竞争对手分析
├── macro_analysis.md            # 宏观环境分析
└── key_findings.md              # 关键发现汇总
```

## Phase 3: 可视化

### 输入
- Phase 1 数据集
- Phase 2 分析报告

### 执行步骤

#### Step 3.1: 调用 data 技能生成图表

**3.1.1 营收趋势图（5年）**
```
调用: data
任务: 创建营收趋势图
参数:
  - 图表类型: line chart / area chart
  - 数据: income.json (营业收入, 营业成本, 营业利润)
  - 时间范围: 最近5年
  - 输出格式: PNG (300 DPI)
输出: revenue_trend.png
```

**3.1.2 利润结构图**
```
调用: data
任务: 创建利润结构图
参数:
  - 图表类型: stacked bar chart / waterfall chart
  - 数据: income.json (毛利, 营业利润, 净利润构成)
  - 输出格式: PNG
输出: profit_structure.png
```

**3.1.3 资产负债结构图**
```
调用: data
任务: 创建资产负债结构图
参数:
  - 图表类型: pie chart / treemap
  - 数据: balance_sheet.json (资产构成, 负债构成)
  - 输出格式: PNG
输出: balance_structure.png
```

**3.1.4 现金流分析图**
```
调用: data
任务: 创建现金流分析图
参数:
  - 图表类型: bar chart (正负柱状图)
  - 数据: cashflow.json (经营/投资/筹资现金流)
  - 输出格式: PNG
输出: cashflow_analysis.png
```

**3.1.5 财务指标对比图（与行业对比）**
```
调用: data
任务: 创建行业对比雷达图
参数:
  - 图表类型: radar chart
  - 数据: 公司vs行业平均 (ROE, ROA, 毛利率, 净利率, 周转率)
  - 输出格式: PNG
输出: industry_comparison.png
```

**3.1.6 竞品对比图**
```
调用: data
任务: 创建竞品对比图
参数:
  - 图表类型: grouped bar chart
  - 数据: 公司与3-5家竞争对手关键指标对比
  - 输出格式: PNG
输出: competitor_comparison.png
```

### 输出

**图表文件清单：**
```
/visualizations/
├── revenue_trend.png            # 营收趋势图
├── profit_structure.png         # 利润结构图
├── balance_structure.png        # 资产负债结构图
├── cashflow_analysis.png        # 现金流分析图
├── industry_comparison.png      # 行业对比雷达图
├── competitor_comparison.png    # 竞品对比图
└── chart_descriptions.md        # 图表说明文档
```

## Phase 4: 内容撰写

### 输入
- Phase 2 分析报告
- Phase 3 图表文件

### 执行步骤

#### Step 4.1: 调用 Word/PDF 技能撰写文档

**4.1.1 按照8大章节结构撰写**

```
调用: word（或 pdf）
任务: 生成商业计划书
章节结构:
  1. 执行摘要
  2. 公司概况
  3. 市场分析
  4. 产品与服务
  5. 商业模式
  6. 财务分析
  7. 团队介绍
  8. 发展规划与融资需求

参数:
  - 输入: /analysis/ 目录下所有分析报告
  - 图表路径: /visualizations/
  - 模板: business_plan_template.docx
  - 格式要求: 专业排版, 页眉页脚, 目录
```

**4.1.2 插入图表**
- 在对应章节插入相关图表
- 添加图表标题和来源标注
- 确保图表编号连续

**4.1.3 添加数据来源标注**
- 所有数据标注来源
- 财务数据标注"数据来源: Tushare Pro"
- 行业数据标注具体来源

**4.1.4 确保每个章节深度达标**
- 对照 chapter-guides.md 检查各章节内容
- 确保分析深度符合用户需求

#### Step 4.2: 调用 Humanizer 技能润色

```
调用: humanizer
任务: 去除AI味，提升可读性
参数:
  - 输入文件: business_plan_draft.docx
  - 处理范围: 全文
  - 风格要求: 专业商业文档风格
输出: business_plan_polished.docx
```

**润色要点：**
- 消除AI生成的套话和重复句式
- 增强逻辑连贯性
- 提升专业术语使用准确性
- 确保语气一致

### 输出

**文档文件：**
```
/output/
├── business_plan_draft.docx     # 初稿
├── business_plan_polished.docx  # 润色后版本
└── business_plan_final.[docx|pdf]  # 最终版本
```

## Phase 5: 质量检查与交付

### 输入
- Phase 4 生成的商业计划书

### 执行步骤

#### Step 5.1: 调用 quality-checklist.md 进行自检

```
读取: quality-checklist.md
执行: 逐项检查
```

**检查清单：**

| 检查项 | 标准 | 状态 |
|--------|------|------|
| 数据完整性 | 最近5年财务数据完整 | ☐ |
| 图表质量 | 所有图表清晰、有标题、有来源 | ☐ |
| 章节结构 | 8大章节完整无缺 | ☐ |
| 分析深度 | 符合用户要求的分析深度 | ☐ |
| 数据来源 | 所有数据标注来源 | ☐ |
| 格式规范 | 专业排版，页眉页脚完整 | ☐ |
| 无AI痕迹 | 无明显AI生成痕迹 | ☐ |

#### Step 5.2: 检查"打回重写"红线

**关键红线（触发任一即打回）：**

- [ ] 数据造假或来源不明
- [ ] 财务计算错误
- [ ] 关键章节缺失（执行摘要、财务分析）
- [ ] 图表与数据不符
- [ ] 明显的AI套话（如"综上所述"等过度使用）
- [ ] 专业术语错误

#### Step 5.3: 修复问题

根据检查结果：
1. 标注所有问题
2. 返回对应Phase修复
3. 修复后重新检查

#### Step 5.4: 最终交付

**交付物：**
- 商业计划书最终文件（Word/PDF）
- 数据来源说明文档
- 图表源文件包

## 错误处理流程

### Phase 1: 需求确认与数据收集

| 错误场景 | 处理策略 |
|----------|----------|
| 数据获取失败 | 1. 使用备选数据源（如从网页抓取）<br>2. 仍失败则向用户汇报，请求手动提供数据 |
| 股票代码错误 | 1. 尝试模糊匹配<br>2. 向用户确认正确代码 |
| 数据不完整 | 1. 标注缺失数据<br>2. 降低该维度分析权重 |

### Phase 2: 财务分析

| 错误场景 | 处理策略 |
|----------|----------|
| 分析失败 | 1. 简化分析维度（如只分析核心指标）<br>2. 标注数据局限性 |
| 计算结果异常 | 1. 人工复核计算公式<br>2. 标注异常原因 |
| Web搜索无结果 | 1. 调整搜索关键词<br>2. 使用备用信息源 |

### Phase 3: 可视化

| 错误场景 | 处理策略 |
|----------|----------|
| 图表生成失败 | 1. 使用替代图表类型<br>2. 仍失败则用文字表格替代 |
| 图表不清晰 | 1. 调整图表参数<br>2. 重新生成高分辨率版本 |
| 数据不匹配 | 1. 核对数据源<br>2. 修正后重新生成 |

### Phase 4: 内容撰写

| 错误场景 | 处理策略 |
|----------|----------|
| 文档生成失败 | 1. 尝试另一种格式（Word转PDF或反之）<br>2. 仍失败则向用户汇报 |
| 图表插入失败 | 1. 检查图表路径<br>2. 手动插入图表 |
| 格式错乱 | 1. 重新应用模板<br>2. 手动调整格式 |

### Phase 5: 质量检查

| 错误场景 | 处理策略 |
|----------|----------|
| 质量检查不通过 | 1. 标注具体问题<br>2. 返回对应Phase修改<br>3. 循环检查直到通过 |
| 关键红线触发 | 1. 立即打回<br>2. 详细说明问题<br>3. 重新执行对应Phase |
