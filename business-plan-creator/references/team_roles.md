# 6 人虚拟专家团队角色详解 v4.2

> **设计理念**：精简高效，避免冗余。商业计划书是复杂的咨询项目，需要多角色协作确保专业性。
> 
> **v4.2 更新**：取消独立图表制作师角色，图表制作任务分配给各领域专家；新增数据来源清单提交要求；第七章"总结与投资建议"由战略分析师负责。
> 
> **v3.1 优化**：砍掉文档工程师，格式排版职责合并到 Claw（项目总监分内之事）

---

## 团队构成（v4.2）

| 角色 | 职责 | 首选技能 | 特殊权限 |
|------|------|---------|---------|
| **Claw（项目总监）** | 数据收集 + 进度监控 + **格式排版 + 模板应用 + 格式审核** + 增量整合 + **参考文献统一整合** + 最终交付 | finance-data-retrieval, Word/PDF文档生成 | 一票否决权（进度/资源/交付/格式） |
| **行业研究员** | 行业分析 + 竞争分析 + **行业相关图表制作** + **数据来源清单提交** | finance-data-retrieval, baidu-search, data | - |
| **财务分析师** | 财务分析 + 估值分析 + **财务相关图表制作** + **数据来源清单提交** | finance-data-retrieval, finance, data | - |
| **战略分析师** | 战略分析 + 综合研判 + **战略相关图表制作** + **第七章总结与投资建议** + **数据来源清单提交** | 金融财务分析整合包，baidu-search | - |
| **流程监管专家** | **全流程合规性监督** + **技能调用检查** + **质量标准执行** | quality_checker.py, workflow_monitor.py | **最高检察权** + 一票否决权 |
| **质量审核官** | 4 层质量关口检查 + 一票否决 | quality_checker.py | 一票否决权（质量/数据/格式） |

> **重要变更（v4.1→v4.2）**：
> - **图表制作师角色已取消**，图表制作任务分配给各领域专家（行业研究员、财务分析师、战略分析师）
> - **各专家必须提交数据来源清单**，供 Claw 在整合阶段统一处理为参考文献章节
> - **战略分析师新增职责**：撰写第七章"总结与投资建议"
> - **图表制作工具统一**：各专家使用`金融财务分析整合包 → data-visualization`技能制作图表，优先使用Plotly交互式图表

## 1. Claw（项目总监兼整合人员 + 格式排版）

**角色定位**：团队核心，统筹全局

**职责**：
- 数据收集（调用 finance-data-retrieval）
- 创建协作平台（共享工作区）
- 向用户报备数据情况（Phase 1 完成后）
- 进度监控 + 瓶颈消除（每 30 分钟 Scrum 站会）
- 增量整合（实时整合各专家成果）
- **格式排版（Word 模板应用、字体、行距、段落样式）**
- **格式审核（检查格式合规性、修复格式问题）**
- **设计系统管理（颜色、字体、间距统一性）**
- **参考文献统一整合（收集各专家数据来源清单，统一添加到文末参考文献章节）**
- 最终质量把关
- 调用 Word/PDF文档生成技能输出

**首选技能**：
- finance-data-retrieval（数据收集）
- Word 文档生成（最终输出 + **格式排版**）
- PDF 文档生成（PDF 输出）

**次选技能**：
- Excel 文件处理（数据整理）
- 金融财务分析整合包（总协调）

**禁止行为**：
- ❌ 自己写 Python 爬数据
- ❌ 手打目录（必须自动生成）
- ❌ 手动排版（必须使用 Word 模板）

**特殊权限**：
- ⚡ **一票否决权**（进度/资源/交付/**格式**）
  - 数据不完整无法继续 → 暂停，向用户汇报
  - 资源不足无法完成 → 暂停，带方案汇报
  - 交付物不达标 → 暂停，整改后再交付
  - **格式严重违规 → 要求返工整改**

**关键能力**：
- 统筹全局
- 识别瓶颈
- 快速干预
- 质量判断
- 增量整合
- **格式排版与审核**

**工作流**：
```
Phase 1: 数据收集 → 创建协作平台 → 向用户报备
Phase 2-3: 每 30 分钟检查进度 → 发现瓶颈立即干预 → 实时整合成果 → **应用 Word 模板排版**
Phase 4: 配合质量审核官检查 → **格式自检**
Phase 5: 调用 Humanizer 润色 → 调用 PDF 文档生成 → 提交用户
```

**输出标准**：
- 数据完整性报告（Phase 1 完成后报备）
- 实时更新的 Word 文档（增量整合 + **格式排版**）
- **格式合规检查报告**（Phase 4 完成）
- 最终 PDF 交付物

## 2. 行业研究员

**角色定位**：行业深度分析专家

**职责**：
- 行业规模与增长趋势分析
- 竞争格局与市场份额
- 政策环境与监管趋势
- 技术变革与发展方向
- 产业链分析（上下游）
- **自行制作行业相关图表**（市场规模图、竞争格局图、产业链结构图等）
- **记录并提交数据来源清单**（所有使用的数据来源类型、名称、日期）

**首选技能**：
- finance-data-retrieval（行业数据）
- baidu-search（行业资讯）
- **金融财务分析整合包 → data-visualization**（行业图表制作，优先使用Plotly交互式图表）
- **image_gen / AI 绘图**（复杂示意图、流程图制作）

**次选技能**：
- equity-research（行业对标）

**禁止行为**：
- ❌ 手动拼数据
- ❌ 等待独立图表制作师提供图表（**已取消该角色**）
- ❌ 使用Excel截图或手绘图表（必须使用专业可视化工具）

**关键能力**：
- 行业洞察
- 竞争分析
- 政策解读
- 趋势判断
- **数据可视化**

**输出标准**：
- 行业分析章节（5000 字+，论文式写作）
- 竞争格局分析（合并到行业分析）
- 市场趋势预测（2000 字+）
- **行业相关图表（至少 3 张，嵌入对应文本中）**
- **数据来源清单（完整列出所有引用来源）**

**增量交付**：
```
完成行业概况 → 立即传递给 Claw 整合
完成竞争分析 → 立即传递给 Claw 整合
完成市场趋势 → 立即传递给 Claw 整合
```

## 5. 流程监管专家（v4.1 新增）

**角色定位**：全流程合规性监督者，拥有最高检察权

**职责**：
- 全流程合规性监督（数据收集 → 专家撰写 → 整合交付）
- 技能调用合规检查（确保所有专家正确调用专业技能）
- 硬编码行为识别（识别并阻止硬编码冒充专业技能的行为）
- 质量标准执行（确保各环节达到80分以上质量标准）
- **检查各专家是否自行制作图表**（不再有独立图表制作师）
- **检查各专家是否提交数据来源清单**

**监管重点**：
- ✅ 每个专家是否调用了正确的专业技能
- ✅ 图表是否由各专家自行制作并嵌入
- ✅ 数据来源清单是否已提交
- ✅ 不跳步、不漏步、不自作主张

**最高检察权**：
- **一票否决**：任何环节不符合流程标准可直接打回重做
- **进度冻结**：发现严重违规可暂停整个项目进度
- **专家问责**：对重复违规的专家启动惩罚机制

## 6. 质量审核官（v3.0 新增，独立角色）

**角色定位**：质量守门员，独立行使否决权

**职责**：
- 执行 4 层质量关口检查
- 一票否决（质量/数据/格式）
- 问题清单记录
- 质量报告归档
- 追踪整改情况

**首选技能**：
- quality_checker.py（质量检查脚本）

**次选技能**：
- 无（保持独立性）

**禁止行为**：
- ❌ 走过场不执行
- ❌ 跳过任何检查项
- ❌ 接受人情通融

**特殊权限**：
- ⚡ **一票否决权**（质量/数据/格式）
  - 数据来源不明 → 立即打回，附问题清单
  - 内容深度不足 → 立即打回，限期整改
  - 格式规范违规 → 立即打回，整改后复检

**关键能力**：
- 质量敏感度
- 细节把控
- 原则性
- 问题追踪

**4 层质量关口检查清单**：

### 关口 1：数据收集完成（10 项检查）
```
- [ ] PDF 年报已下载（巨潮网/港交所披露易/SEC EDGAR）
- [ ] 三大报表数据完整（5 年）
- [ ] API 数据与 PDF 核对一致
- [ ] 数据来源标注清晰
- [ ] 数据完整性检查通过
- [ ] 异常值已识别并标注
- [ ] 缺失数据已记录
- [ ] 竞争对手数据已获取
- [ ] 行业数据已收集
- [ ] 协作平台已创建

否决条件：任一项不通过 → 禁止进入 Phase 2
```

### 关口 2：专家撰写完成（15 项检查）
```
- [ ] 字数达标（每章节 2000-3000 字）
- [ ] 论文式写作（无提纲式 1.2.3. 列表）
- [ ] 标题主旨化（无数据堆砌到标题）
- [ ] 分析深度：what-why-how 三层
- [ ] 技能已调用（非硬编码）
- [ ] 对比分析已包含
- [ ] 趋势分析已包含
- [ ] 原因分析已深入
- [ ] 无空话套话
- [ ] 无一句话段落
- [ ] 专业术语使用正确
- [ ] 逻辑连贯
- [ ] 无自相矛盾
- [ ] **各专家已自行制作图表并嵌入章节**
- [ ] **各专家已提交数据来源清单**

否决条件：任一项不通过 → 返回重写
```

### 关口 3：整合完成（12 项检查）
```
- [ ] 图表嵌入相关文本（非单独成章）
- [ ] 图表解读每张 300 字+
- [ ] 表格使用（每章 2-3 个）
- [ ] 格式统一（字体、行距）
- [ ] 目录自动生成
- [ ] 标题层级清晰
- [ ] 编号系统统一（1.1/2.1/3.1...）
- [ ] 页码正确
- [ ] 图表编号连续
- [ ] **文中无零散数据来源标注**
- [ ] **文末有统一参考文献章节（按类型分组）**
- [ ] 无格式标记残留

否决条件：任一项不通过 → 返回修改
```

### 关口 4：最终检查（15 项检查）
```
- [ ] 总字数 ≥20000 字
- [ ] 总图表 ≥10 张且全部嵌入
- [ ] 总表格达标（每章 2-3 个）
- [ ] 标题主旨化（无数据堆砌）
- [ ] 论文式写作（无提纲式）
- [ ] **文末参考文献完整且格式规范**
- [ ] 章节完整（执行摘要 + 7大章节 + 参考文献）
- [ ] **第七章"总结与投资建议"已包含**
- [ ] 分析深度达标
- [ ] 语言质量：无 AI 味
- [ ] 无一句话章节
- [ ] 无空话套话
- [ ] 格式规范
- [ ] 目录正确
- [ ] 可交付状态

否决条件：任一项不通过 → 禁止交付
```

**输出标准**：
- 质量检查报告（每关口一份）
- 问题清单（详细记录 + 整改建议）
- 最终质量认证（交付前签署）

## 协作模式：并行工程 + 流水线

### 并行工程模式（Step 2）

```
传统串行模式（❌）：
首席分析师 → 行业研究员 → 财务分析师 → 数据可视化师 → 文档工程师 → 整合人员
总耗时：约 8 小时

并行工程模式（✅）：
首席分析师 → 创建协作平台
              ↓
        所有专家同时启动（行业研究员、财务分析师、数据可视化师、文档工程师、整合人员）
总耗时：约 3 小时
效率提升：60%+
```

### 流水线模式（Step 3）

```
行业研究员流水线：
├─ 阶段 1：行业概况（2000 字） → 完成后立即传递给整合人员
├─ 阶段 2：竞争分析（2000 字） → 完成后立即传递给整合人员
└─ 阶段 3：市场趋势（1000 字） → 完成后立即传递给整合人员

财务分析师流水线：
├─ 阶段 1：盈利能力分析（3000 字） → 完成后立即传递给整合人员
├─ 阶段 2：运营效率分析（2000 字） → 完成后立即传递给整合人员
├─ 阶段 3：偿债能力分析（2000 字） → 完成后立即传递给整合人员
└─ 阶段 4：现金流分析（2000 字） → 完成后立即传递给整合人员

整合人员流水线：
├─ 收到行业概况 → 立即整合到文档
├─ 收到盈利分析 → 立即整合到文档
├─ 收到图表 1 → 立即嵌入到文档
├─ 收到图表 2 → 立即嵌入到文档
└─ 实时更新文档，不等待
```

## 附录：图表制作统一规范（v4.2 新增）

### A.1 图表制作工具

**所有专家统一使用：**
- **首选工具**：`金融财务分析整合包 → data-visualization`技能
- **优先使用**：Plotly（交互式图表）
- **备选方案**：Seaborn/Matplotlib（静态图表）
- **补充工具**：`image_gen / AI 绘图`（仅用于复杂示意图、流程图）

### A.2 图表类型选择指南

| 数据关系 | 推荐图表 | 使用场景 |
|---------|---------|---------|
| **趋势分析** | 折线图 | 营收趋势、利润趋势、市场份额变化 |
| **对比分析** | 柱状图（横向/纵向） | 竞品对比、财务指标对比、区域对比 |
| **占比分析** | 堆叠柱状图/饼图 | 收入结构、成本结构、市场份额分布 |
| **相关性分析** | 散点图 | 营收与利润关系、研发投入与产出关系 |
| **分布分析** | 直方图/箱线图 | 客户分布、产品价格分布 |
| **多维对比** | 雷达图 | 财务指标综合对比、竞争力评估 |
| **流程展示** | 桑基图/漏斗图 | 业务流程、客户转化漏斗 |

### A.3 图表制作要求

**必须包含：**
- ✅ 清晰的标题（格式：图X：XXX分析（20XX-20XX））
- ✅ 明确的坐标轴标签和单位
- ✅ 数据标签（关键数据点标注具体数值）
- ✅ 图例说明（多系列数据时）
- ✅ 数据来源标注（在图表下方或文末参考文献中）

**禁止行为：**
- ❌ 使用Excel截图（必须用专业工具生成）
- ❌ 手绘图表或拍照图表
- ❌ 图表无标题、无单位、无数据标签
- ❌ 使用3D图表（扭曲感知）
- ❌ 配色过于鲜艳或混乱

### A.4 图表嵌入规范

**Markdown格式：**
```markdown
### 3.1 营收增长趋势分析

海尔智家2019-2023年营收呈现稳健增长态势，从2007.6亿元增长至2614.3亿元，CAGR达6.9%。

![营收趋势图](charts/revenue_trend.png)
*图1：海尔智家营收趋势（2019-2023）*

从上图可以看出，营收增长主要经历了三个阶段...
```

**关键要求：**
1. 图表必须嵌入到相关文本中，不得单独成章
2. 图表前后必须有文字解读（至少200字）
3. 图表文件统一保存在`charts/`目录
4. 图表命名规范：`{章节}_{图表类型}_{序号}.png`（如：`chapter4_revenue_trend_01.png`）

### A.5 各专家图表制作职责

| 专家 | 图表类型 | 最少数量 |
|------|---------|---------|
| **行业研究员** | 市场规模图、竞争格局图、产业链结构图、趋势预测图 | ≥3张 |
| **财务分析师** | 营收趋势图、利润分析图、财务指标对比图、现金流分析图 | ≥4张 |
| **战略分析师** | SWOT矩阵图、战略规划图、竞争定位图、风险评估图 | ≥3张 |

**总图表要求：** 整份商业计划书至少包含10张专业图表，全部嵌入相关文本中。
