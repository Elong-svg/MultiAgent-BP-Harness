# business-plan-creator 商业计划书生成器 v4.0

> **核心理念**：通过虚拟专家团队（5 人）+ 专业子技能（3 个）+ Claw 协作模式，生成内容深入、数据可追溯、专业水准的商业计划书。
> 
> **v4.0 核心升级**：新增三个专业子技能，形成完整工作流
> - **data-cleaner** - 数据清洗引擎（Step 1 之后自动调用）
> - **business-writer** - 商业写作引擎（Step 2 专家撰写时调用）
> - **document-integrator** - 文档整合引擎（Step 3 整合阶段调用）

## 快速启动

```bash
# 用户触发：写商业计划书
# 自动启动：5 人虚拟专家团队 + Claw → 并行工程模式 → 增量交付 → 4 层质量关口 → 交付
```

## 核心架构：5 人虚拟专家团队 + Claw + 质量审核官

| 角色 | 职责 | 首选技能 | 次选技能 | 禁止行为 | 特殊权限 |
|------|------|---------|---------|---------|---------|
| **Claw（项目总监）** | 数据收集 + 进度监控 + **格式排版 + 模板应用 + 格式审核** + 质量把关 + 增量整合 | finance-data-retrieval, Word/PDF文档生成 | Excel文件处理, 金融财务分析整合包 | ❌ 自己写Python爬数据 | **一票否决权**（进度/资源/交付/**格式**） |
| **行业研究员** | 行业分析 + 竞争分析 | finance-data-retrieval, baidu-search, data | equity-research | ❌ 手动拼数据 | - |
| **财务分析师** | 财务分析 + 估值分析 | finance-data-retrieval, finance, data | Excel文件处理, 金融财务分析整合包 | ❌ 手算财务指标 | - |
| **战略分析师** | 战略分析 + 综合研判 | 金融财务分析整合包, baidu-search | 市场调研 | ❌ 纯理论不结合数据 | - |
| **图表制作师** | 图表生成 + 图表解读 | data, Excel文件处理, AI绘图 | - | ❌ 用emoji当图标 | - |
| **质量审核官** | 4层质量关口检查 + 一票否决 | quality_checker.py | - | ❌ 跳过任何检查项 | **一票否决权**（质量/数据/格式） |

**设计理念**：
- 每个角色**强制绑定技能**，禁止硬编码替代
- Claw 兼任整合人员，负责增量整合和实时更新
- 质量审核官独立于团队，有**一票否决权**
- 5 人团队精简高效，避免角色冗余

## 工作流程（并行工程模式）

```
Step 1: Claw → 数据收集（PDF优先）+ 创建协作平台 → 向用户报备
Step 2: 所有专家同时启动（并行工程！无等待）
Step 3: 增量交付 + Claw 实时整合（流水线模式）
Step 4: 进度监控 + 瓶颈消除（Scrum 站会）
Step 5: 4 层质量关口 + 迭代改进（敏捷 Sprint）
```

详细流程 → 见 `references/workflow_guide.md`

## 三条铁律（v3.0 核心）

### 铁律一：数据溯源
- ✅ **PDF 财报优先** - 上市公司年报 PDF 为一手数据源
- ✅ **API 数据核对** - API 数据需与 PDF 核对一致
- ✅ **来源标注** - 所有数据标注具体来源（年报页码/数据源）
- ❌ 禁止网上零散数据堆砌

### 铁律二：论文式写作
- ✅ **段落论述** - 每个分析点用完整段落论述
- ❌ **禁止提纲式** - 禁止 1.2.3. 列表式写作
- ❌ **禁止一句话章节** - 每章节需深入分析

### 铁律三：标题主旨化
- ✅ **标题 = 分析角度** - 如"3.1 营收增长趋势分析"
- ❌ **禁止数据堆砌到标题** - 如"3.1 营收五年复合增长 18%"
- ✅ **数据在正文中呈现** - 标题体现要分析什么

## 关键规则（铁律）

### 内容质量铁律
- ✅ **What-Why-How 三层分析** - 禁止只罗列数据
- ✅ **数据可追溯** - 所有数据标注来源
- ✅ **财务分析五大维度** - 盈利能力、运营效率、偿债能力、成长能力、现金流
- ✅ **图表必须嵌入** - 每张图表配 300 字 + 解读，嵌入相关文本
- ✅ **段落式论述** - 禁止 1.2.3.列表化
- ✅ **表格使用** - 每章至少 2-3 个表格，用于对比、总结、归纳

### 流程铁律
- ✅ **Step 1 必须先报备** - 数据收集完成后向用户报备
- ✅ **并行工程模式** - 所有专家同时启动，无等待
- ✅ **增量交付** - 完成一部分立即传递
- ✅ **4 层质量关口** - 每层有否决权

### 禁止事项
- ❌ 用 Python 硬编码 Word（必须调用 Word 文档生成技能）
- ❌ 手打目录（必须自动生成）
- ❌ 只罗列不分析
- ❌ 图表和文字分离
- ❌ 网上零散数据堆砌
- ❌ 提纲式写作（1.2.3.列表）
- ❌ 标题堆砌数据

## 质量关口（4 层否决机制）

### 关口 1：数据收集完成
```
检查项（10 项）：
- [ ] PDF 年报已下载（巨潮网/港交所披露易/SEC EDGAR）
- [ ] 三大报表数据完整（5 年）
- [ ] API 数据与 PDF 核对一致
- [ ] 数据来源标注清晰
- [ ] 数据完整性检查通过
- [ ] 异常值已识别并标注
- [ ] 缺失数据已记录
- [ ] 竞争对手数据已获取
- [ ] 行业数据已收集
- [ ] 协作平台已创建

否决条件：任一项不通过 → 禁止进入 Step 2
```

### 关口 2：专家撰写完成
```
检查项（15 项）：
- [ ] 字数达标（每章节 2000-3000 字）
- [ ] 论文式写作（无提纲式 1.2.3. 列表）
- [ ] 标题主旨化（无数据堆砌到标题）
- [ ] 分析深度：what-why-how 三层
- [ ] 技能已调用（非硬编码）
- [ ] 数据来源已标注
- [ ] 对比分析已包含
- [ ] 趋势分析已包含
- [ ] 原因分析已深入
- [ ] 无空话套话
- [ ] 无一句话段落
- [ ] 专业术语使用正确
- [ ] 逻辑连贯
- [ ] 无自相矛盾
- [ ] 图表已规划

否决条件：任一项不通过 → 返回重写
```

### 关口 3：整合完成
```
检查项（12 项）：
- [ ] 图表嵌入相关文本（非单独成章）
- [ ] 图表解读每张 300 字+
- [ ] 表格使用（每章 2-3 个）
- [ ] 格式统一（字体、行距）
- [ ] 目录自动生成
- [ ] 标题层级清晰
- [ ] 编号系统统一（1.1/2.1/3.1...）
- [ ] 页码正确
- [ ] 图表编号连续
- [ ] 数据来源完整
- [ ] 无格式标记残留
- [ ] 无提示词泄露

否决条件：任一项不通过 → 返回修改
```

### 关口 4：最终检查
```
检查项（15 项）：
- [ ] 总字数 ≥20000 字
- [ ] 总图表 ≥10 张且全部嵌入
- [ ] 总表格达标（每章 2-3 个）
- [ ] 标题主旨化（无数据堆砌）
- [ ] 论文式写作（无提纲式）
- [ ] 数据来源标注完整
- [ ] 章节完整（8 大章节）
- [ ] 分析深度达标
- [ ] 语言质量：无 AI 味
- [ ] 无一句话章节
- [ ] 无空话套话
- [ ] 格式规范
- [ ] 目录正确
- [ ] 页码正确
- [ ] 可交付状态

否决条件：任一项不通过 → 禁止交付
```

## 双重否决机制

| 角色 | 否决范围 | 否决触发条件 | 否决后处理 |
|------|---------|-------------|-----------|
| **项目总监（Claw）** | 进度/资源/交付 | - 数据不完整无法继续<br>- 资源不足无法完成<br>- 交付物不达标 | 立即暂停，向用户汇报，带方案 |
| **质量审核官** | 质量/数据/格式 | - 数据来源不明<br>- 内容深度不足<br>- 格式规范违规 | 立即打回，附问题清单，限期整改 |
