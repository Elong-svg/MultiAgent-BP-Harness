# 商业计划书完整工作流程指南 v4.0

> **v4.0 更新**：新增三个专业子技能（data-cleaner、business-writer、document-integrator），形成完整 10 技能工作流

> **核心模式**：并行工程 + 流水线模式 + Scrum 站会 + 敏捷 Sprint

---

## 总览：5 步法

```
Step 1: Claw（项目总监） → 数据收集 + 创建协作平台 → 向用户报备
Step 2: 所有专家同时启动（并行工程模式 - 无等待！）
Step 3: 增量交付 + 实时整合（流水线模式）
Step 4: 进度监控 + 瓶颈消除（Scrum 站会模式）
Step 5: 质量关口 + 迭代改进（敏捷 Sprint 模式）
```

**效率提升**：
- 等待时间：减少 50%
- 整合时间：减少 70%
- 发现问题：提前 90%
- 修改成本：减少 80%
- 协作效率：提升 200%

---

## Step 1: 数据收集 + 创建协作平台

### 1.1 数据收集

**负责人**：Claw（项目总监）

**动作**：
```python
# 调用 finance-data-retrieval 技能
use_skill: finance-data-retrieval

# 获取数据清单
- 年报 PDF（巨潮网）
- 利润表（2019-2023 年）
- 资产负债表（2019-2023 年）
- 现金流量表（2019-2023 年）
- 财务指标（ROE、毛利率、资产负债率等 20+ 指标）
```

**关键要求**：
- ✅ 必须使用巨潮网官方 PDF 一手数据
- ✅ 数据完整性：三大报表齐全
- ✅ 数据来源：标注清晰
- ✅ 数据准确性：API 数据与 PDF 数据核对一致
- ✅ **调用 data-cleaner 清洗数据**（v4.0 新增）

### 1.2 创建协作平台

**动作**：
```python
# 创建团队
team_create: {
  "team_name": "bp-[公司]-[日期]",
  "description": "商业计划书协作平台 - 并行工程模式"
}

# 创建共享工作区目录
mkdir: .workbuddy/teams/bp-[公司]-[日期]/
  ├── data/        # 共享数据区
  ├── drafts/      # 草稿区
  ├── charts/      # 图表区
  ├── progress/    # 进度追踪
  └── quality/     # 质量关口
```

**目录说明**：
- `data/` - 存放财务数据、行业数据（所有专家实时访问）
- `drafts/` - 各专家草稿（增量交付）
- `charts/` - 生成的图表（PNG 格式）
- `progress/` - 进度追踪文件（status.json）
- `quality/` - 质量关口检查记录

### 1.3 向用户报备

**关键要求**：
- ⚠️ **必须先报备** - 用户确认后才能进入 Step 2
- ⚠️ **数据不完整禁止进入下一阶段**

## Step 2: 所有专家同时启动（并行工程模式）

### 2.1 团队构成（v4.0 更新版）

| 角色 | 职责 | 输出 | 强制绑定技能 |
|------|------|------|-------------|
| **Claw（项目总监）** | 进度监控 + 质量把关 + 格式排版 | 统筹全局 | finance-data-retrieval, data-cleaner, Word/PDF |
| **行业研究员** | 行业 + 竞争分析 | 5000 字 + | finance-data-retrieval, baidu-search, **business-writer** |
| **财务分析师** | 财务 + 估值分析 | 10000 字 + | finance-data-retrieval, finance, **business-writer** |
| **战略分析师** | 战略 + 综合研判 | 5000 字 + | 金融财务分析整合包，**business-writer** |
| **图表制作师** | 图表 + 解读 | 10 张图表 | data, Excel, AI 绘图 |
| **质量审核官** | 4 层质量关口检查 | 质量评分 | quality_checker.py |

**v4.0 核心升级**：
- ✅ **data-cleaner** - Step 1 数据收集后自动调用，清洗数据并评级
- ✅ **business-writer** - Step 2 专家撰写时强制调用，确保写作质量统一
- ✅ **document-integrator** - Step 3 整合阶段调用，精准定位图表并统一格式

### 2.2 并行启动（关键！）

**❌ 旧流程（串行）**：
```
首席分析师 → 行业研究员 → 财务分析师 → 数据可视化师 → 文档工程师 → 整合人员
总耗时：约 8 小时
```

**✅ 新流程（并行）**：
```
首席分析师 → 创建协作平台
              ↓
        所有专家同时启动！
        ├── 行业研究员
        ├── 财务分析师
        ├── 数据可视化师
        ├── 文档工程师
        └── 整合人员
总耗时：约 3 小时
效率提升：60%+
```


```
│  └─ 完成后立即传递给整合人员

   └─ 完成后立即传递给整合人员
```

**财务分析师流水线**：
```
├─ 阶段 1：盈利能力分析（3000 字）

│  └─ 完成后立即传递给整合人员
├─ 阶段 3：偿债能力分析（2000 字）
│  └─ 完成后立即传递给整合人员
└─ 阶段 4：现金流分析（2000 字）
   └─ 完成后立即传递给整合人员
```

**数据可视化师流水线**：
```
├─ 阶段 1：图表 1-3 + 解读
│  └─ 完成后立即传递给整合人员
├─ 阶段 2：图表 4-6 + 解读
│  └─ 完成后立即传递给整合人员
└─ 阶段 3：图表 7-10 + 解读

```

```
├─ 收到图表 1 解读.md → 立即添加到图表下方
└─ 实时更新文档，不等待所有专家完成
```

## Step 5: 质量关口 + 迭代改进（敏捷 Sprint）

### 5.1 质量关口设置

**关口 1：数据收集完成**
```
检查项：
- [ ] 数据完整性：三大报表齐全
- [ ] 数据来源：标注清晰
- [ ] 数据准确性：API 与 PDF 一致

通过 → 进入 Step 2
不通过 → 补充数据
```

**关口 2：专家撰写完成**
```
检查项：
- [ ] 字数达标：每章节 2000-3000 字
- [ ] 分析深度：what-why-how 三层
- [ ] 数据解读：有解读非罗列

通过 → 进入整合
不通过 → 返回修改
```

**关口 3：整合完成**
```
检查项：
- [ ] 图表嵌入：所有图表已嵌入
- [ ] 图表解读：每张 300 字+
- [ ] 格式统一：字体、行距一致
- [ ] 目录生成：自动生成

通过 → 进入最终检查
不通过 → 返回修改
```

**关口 4：最终检查**
```
检查项：
- [ ] 总字数：≥20000 字
- [ ] 总图表：≥10 张且全部嵌入
- [ ] 章节完整：8 大章节齐全
- [ ] 分析深度：what-why-how
- [ ] 数据来源：标注完整
- [ ] 语言质量：无 AI 味

通过 → 提交用户
不通过 → 返回修改
```

### 5.2 Sprint 周期

```
Sprint 1（数据收集）：30 分钟
  └─ 完成数据收集
  └─ 关口 1 检查
  └─ 通过 → 进入 Sprint 2

Sprint 2（专家撰写）：2 小时
  └─ 所有专家并行撰写
  └─ 关口 2 检查（增量）
  └─ 通过 → 进入 Sprint 3

Sprint 3（整合）：1 小时
  └─ 增量整合
  └─ 关口 3 检查
  └─ 通过 → 进入 Sprint 4

Sprint 4（最终检查）：30 分钟
  └─ 最终质量检查
  └─ 关口 4 检查
  └─ 通过 → 提交用户
```

### 5.3 迭代改进机制

**每个 Sprint 结束后**：
1. 检查关口通过率
2. 识别问题点
3. 制定改进措施
4. 应用到下一个 Sprint

**示例**：
```
Sprint 2 发现问题：财务分析师盈利分析卡住
改进措施：提供盈利分析模板 + 指标解读指南
应用到 Sprint 3：财务分析师效率提升 40%
```
