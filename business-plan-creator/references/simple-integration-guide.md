# 简化整合流程指南（v4.2）

> **核心理念**：直接拼接 + 统一后处理，杜绝重写和污染

## 一、整合原则

### 1.1 零重写原则
- **绝不重写**专家撰写的任何内容
- **原封不动**保留专家的分析、论述、图片嵌入
- **直接拼接**各专家文档，不做内容修改

### 1.2 零污染原则  
- **移除文中所有数据来源标注**
- **统一在文章末尾添加参考文献**
- **保持正文纯净，专注内容本身**

### 1.3 专业输出原则
- **调用专业文档生成技能**（Word/PDF）
- **不自行硬编码**任何格式处理
- **依赖专业工具**处理排版、目录、格式

## 二、整合流程（3步法）

### Step 1: 直接拼接
```
输入：N个专家文档（Markdown格式）
处理：按顺序直接拼接
输出：初步整合文档
```

**操作要点**：
- 按专家交付顺序拼接：执行摘要 → 公司概况 → 行业分析 → 财务分析 → 战略分析
- 不做任何内容修改，包括标点、用词、段落结构
- 保留所有图片嵌入：`![描述](charts/filename.png)`

### Step 2: 统一后处理
```
输入：初步整合文档
处理：标题规范化 + 数据来源统一 + 整体性检查  
输出：标准化整合文档
```

**具体操作**：

#### 2.1 标题规范化
- **统一编号系统**：1.1, 1.2, 2.1, 2.2, 3.1...
- **章节标题格式**：
  - 一级标题：`# 第X章 章节名称`
  - 二级标题：`## X.Y 小节名称`
  - 三级标题：`### X.Y.Z 子小节名称`

#### 2.2 数据来源统一处理（v4.2 强化）
- **收集各专家提交的数据来源清单** - 每个专家在撰写时必须记录引用的所有数据来源
- **扫描并移除**所有文中零散的数据来源标注，如：
  - "（数据来源：公司年报）"
  - "（来源：Wind数据库）"  
  - "（根据Tushare数据）"
- **在文章末尾统一添加**「参考文献」章节：
  ```markdown
  ## 参考文献

  ### 公司官方文件
  - [1] 海尔智家股份有限公司2023年年度报告（2024-03-28）
  - [2] 海尔智家股份有限公司2023年半年度报告（2023-08-30）

  ### 行业研究报告
  - [3] 艾瑞咨询《2023中国智能家居行业研究报告》（2023-06）
  - [4] 中商产业研究院《2023年中国家电行业市场前景分析报告》（2023-03）

  ### 数据库/数据接口
  - [5] Tushare Pro API - 财务指标数据（2024-01）
  - [6] 新浪财经 - 公司基本面数据（2024-01）

  ### 政府公开数据
  - [7] 国家统计局 - 规模以上工业企业利润数据（2024-01）
  ```

**参考文献格式规范**（v4.2 强制）：
- **必须按类型分组**：公司官方文件 > 行业研究报告 > 数据库/数据接口 > 政府公开数据 > 媒体报道
- **每个来源必须包含**：序号、来源名称/标题、日期
- **同类型内按时间倒序排列**
- **禁止编造来源** - 所有来源必须可追溯

#### 2.3 整体性检查
- **章节连贯性**：确保章节间有逻辑衔接
- **重复内容检查**：避免不同专家重复相同内容
- **术语一致性**：统一关键术语（如"营业收入" vs "营收"）

### Step 3: 专业输出
```
输入：标准化整合文档
处理：调用专业文档生成技能
输出：最终商业计划书（Word/PDF）
```

**技能调用**：
- **首选**：Word 文档生成技能
- **备选**：PDF 文档生成技能  
- **严禁**：自行用python-docx等硬编码

## 三、Claw（项目总监）职责

### 3.1 整合阶段职责
| 任务 | 正确做法 | 错误做法 |
|------|---------|---------|
| **文档拼接** | 直接拼接，不做改写 | 重新用自己的话重写 |
| **格式处理** | 调用专业技能处理 | 自己硬编码处理格式 |
| **数据来源** | 统一移到文末 | 在文中到处标注 |
| **质量检查** | 调用质量检查工具 | 凭感觉判断 |

### 3.2 禁止行为清单
- ❌ **重写专家内容**（哪怕觉得写得不好）
- ❌ **自行处理格式**（必须调用专业技能）
- ❌ **保留文中数据来源**（必须统一移到末尾）
- ❌ **跳过专业技能**（必须通过Word/PDF技能输出）

## 四、质量检查清单

### 整合完成检查项（10项）：
- [ ] **专家内容完整保留**（无任何重写）
- [ ] **图片正确嵌入**（路径正确，显示正常）
- [ ] **标题编号统一**（1.1, 2.1, 3.1...格式）
- [ ] **文中无零散数据来源标注**
- [ ] **文末有「参考文献」章节**（按类型分组、格式规范、来源可追溯）
- [ ] **第七章"总结与投资建议"已包含**
- [ ] **章节连贯**（无逻辑断层）
- [ ] **术语一致**（关键术语统一）

## 五、技术实现示例

### Python整合脚本框架：
```python
def simple_integration(expert_docs):
    """简化整合流程"""
    
    # Step 1: 直接拼接
    combined = ""
    for doc in expert_docs:
        combined += doc + "\n\n---\n\n"
    
    # Step 2: 统一后处理
    combined = normalize_headings(combined)           # 标题规范化
    combined = remove_inline_citations(combined)      # 移除文中数据来源
    combined = add_unified_references(combined)       # 添加统一参考文献
    combined = check_consistency(combined)            # 一致性检查
    
    # Step 3: 专业输出（调用技能，不硬编码）
    call_word_document_skill(combined)
    
    return combined
```

### 标题规范化函数：
```python
def normalize_headings(content):
    """统一标题编号"""
    chapters = [
        "执行摘要", "公司概况", "行业分析", 
        "财务分析", "战略分析"
    ]
    
    lines = content.split('\n')
    chapter_index = 0
    section_index = 0
    
    for i, line in enumerate(lines):
        if line.startswith('# ') and not line.startswith('# '):
            # 一级标题
            if chapter_index < len(chapters):
                lines[i] = f"# 第{chapter_index + 1}章 {chapters[chapter_index]}"
                section_index = 1
                chapter_index += 1
        elif line.startswith('## '):
            # 二级标题
            lines[i] = f"## {chapter_index}.{section_index} {line[3:].strip()}"
            section_index += 1
    
    return '\n'.join(lines)
```

## 六、与旧流程对比

| 方面 | 旧流程（document-integrator） | 新流程（简化整合） |
|------|------------------------------|-------------------|
| **复杂度** | 复杂（锚点系统、解析引擎） | 简单（直接拼接） |
| **可靠性** | 低（技能未实现） | 高（直接操作） |
| **专家负担** | 高（需学习锚点语法） | 低（专注内容） |
| **整合质量** | 不稳定（依赖复杂解析） | 稳定（保留原内容） |
| **实施难度** | 高（需要完整技能实现） | 低（简单脚本即可） |
